## 1. サイト全体の構成（確定）

```
トップページ（完成済み）
   ├─ イベント・体験AR ページ        ← このチャットで作る
   ├─ 印刷物・商品AR ページ          ← このチャットで作る
   └─ AR入門コラム                    ← 現行LPの「ARとは？」等の移転先。優先度は低め
   ↓
無料相談フォーム（全ページ共通の出口）
```

※事例詳細ページは今回は作らない（実績がまだ少ないため。各サービスページ内で実績を1〜2件紹介すれば足りる。事例が増えてから独立ページ化する）。

### サービス分類の大原則（最重要・ここを外さない）

- **技術ではなく「お客さんが何をしたいか」で分類する**
- 「マーカーAR」「インタラクティブAR」のような技術名での分類は不可（お客さんが理解できず、自分ごとにならないため。過去に検討して却下済み）
- 2本柱＋カスタム相談枠の構成：
  - 柱1：イベント・体験AR（マルシェ・商業施設・展示会・地域イベント向け）
  - 柱2：印刷物・商品AR（名刺・チラシ・カタログ・パッケージ・CD向け）
  - 相談枠：観光マップ・文化財演出・オリジナル企画など（実績が増えたら3本目の柱に昇格）
- AR CDは柱2に含む。ARフォトフレームは柱1が主（納品物としては柱2にも登場可）

## 2. メッセージング方針（トップと統一する）

- キーワードは「体験」。メインコピーは「お客さんが足を止める『体験』、つくります。」
- 言い換えOK：「感動体験」「驚き（の）体験」。NG：「サプライズ体験」「常識を覆す体験」（広告的誇張＝避けたいAIっぽさ）
- ARという手段ではなく、お客さん側に起きる結果（足を止める・記憶に残る・盛り上がる）を先に語る
- 想定読者はARの知識ゼロのイベント主催者・店舗オーナー。専門用語を使わず、ですます調
- 相談ハードルを下げる言葉を各所に（「思いつきの段階でOK」「予算はこれだけ、の相談も歓迎」）
- 「テンプレートを使わない」とは言わない。テンプレートは引き出しの1つ。「目的に合わせて、テンプレ活用もオリジナル企画も、予算に応じて選べる」という立て付け（competitorとの差別化の核）

### デザイン方針（脱・AIっぽさ。トップと統一）

- 実写の写真・動画を主役に。イラスト・汎用アイコンは脇役
- 絵文字は使わない
- 同じ大きさのカードを等間隔に並べる構成を避け、サイズ・配置に強弱
- 色数を絞る（ベース＋アクセント1色）
- 見出しに「〜とは？」「◯つのメリット」型を使わない
- CTAボタンは主・副の2種類のみ
- モバイルファースト（流入はQR・Instagram経由が中心でスマホ閲覧が大半という推測にもとづく）

## 3. 各下層ページの構成方針（たたき台）

### ■ イベント・体験AR ページ

想定読者：集客に悩むイベント主催者・販促担当者。ARという言葉を知らなくても読める文章にする。
このページの説得力は実写素材（来場者が遊んでいる引きの動画・写真）の量で決まる。素材が少ない場合は、今後のイベント（イオンモール、農協イベント等）で「ページ用素材を撮る」をタスク化する前提。

構成案：
1. ファーストビュー：イベントで人が体験している実写動画＋「集客に悩むイベントに」のような切り口
2. 体験メニュー一覧：Catch The Fruits! / Finger Shot! / ARフォトフレーム（それぞれ動画付き）
3. 3つの提供形態（下記）と料金
4. ご利用の流れ（お申込み〜開催当日まで何日かかるか）
5. 主催者向けFAQ（機材・電源・通信環境・雨天時・対象年齢・同時に何人体験できるか 等）
6. 相談CTA

提供形態（関わりの深さ順）：
- ① 既製レンタル：完成済みコンテンツをそのまま貸出。QR/URL納品、運営は主催者側
- ② カスタマイズレンタル：ロゴ・素材の差し替え。基本カスタマイズが短時間で完了する開発体制が強み＝主力商品
- ③ 企画・出張サポート：新規制作、当日の設営・運営同行まで

### ■ 印刷物・商品AR ページ

想定読者：名刺・チラシ・商品に付加価値をつけたい中小企業・店舗・個人事業主。

構成案：
1. ファーストビュー：「配って終わりの紙を、体験に変える」のような切り口
2. 対応できるもの：名刺・チラシ・カタログ・パッケージ・CD（それぞれ実例があれば動画/写真）
3. 機能による料金の段階表示（※技術名ではなく「できること」で段階を見せる）
   例：かざすと動画が再生 → 3D表示・タップ操作 → 複数マーカー・分岐演出、と機能が増えるほど料金が上がる見せ方
4. ご利用の流れ
5. FAQ
6. 相談CTA

### ■ 実績の見せ方（事例詳細ページは作らない）

- 今回は独立した事例詳細ページを作らない（実績がまだ少ないため）
- 代わりに、各サービスページの中で実績を1〜2件紹介する
  - イベント・体験ARページ：Catch The Fruits! の現場、音羽の歌舞伎AR（※承諾後）など
  - 印刷物・商品ARページ：AR名刺、AR CD など
- 事例が増えてきた段階で、独立した事例ページ／事例一覧を新設する

- 基準は正規価格。キャンペーン価格は割引として従属表示（安価アンカーを防ぐため）
- 最低価格だけでなく「中心価格帯」を併記
- 印刷物・商品ARページには「機能が増えるほど料金が上がる」段階表示を入れ、技術による価格差を顧客目線で表現する

### 公開・サポートの注意書き（各サービスページに掲載する全文）
- WebARは端末のブラウザ・OSに依存し、将来のアップデートで表示が変わる場合がある
- 動作保証は納品後1ヶ月（納品時点の主要ブラウザ・OSでの不具合修正）
- 公開（ホスティング）は継続するが、1ヶ月経過後の動作は保証しない
- OS・ブラウザ更新に伴う修正、内容の変更・差し替えは都度見積り
- データのお渡しを希望する場合は有料対応
- 意図：「一度作れば永遠に動く」という誤解を技術的事実にもとづいて防ぐ

## 5. competitor 分析からわかった独自の強み（下層ページでも効かせる）

市場は2極化していて、その中間が空いている：
- セルフ型ツール（palanAR/COCOAR/LESSAR等）：安いが、お客さん自身が作る必要あり
- 大手・中堅の受託（50万〜、自治体観光ARは250〜400万が相場）：高品質だが高額で地域の小規模案件には手が出ない
- → Tech Connectが狙う「自分では無理。でも数百万は出せない」中間層（地域のマルシェ・商店街・小自治体・個人店）はプレイヤーが少ない空白地帯

独自の強み3点（トップのセクション4で使用。下層でも一貫させる）：
1. 数万円から、小さく始められます（レンタルという気軽な入口。競合は買い切りか月額契約の二択が多い）
2. 目的に合わせて、いちばんいい方法を選びます（テンプレ活用もオリジナルも、予算に応じて。competitorはテンプレ量産かフルカスタムに寄る）
3. 相談から当日まで、私が直接動きます（企画・制作・当日運営を一人で通せる。競合は役割が分断されがち）

※強み2・3（企画から一緒に・当日まで動く）は工数が増える約束でもあるため、主に中〜高単価帯（カスタマイズ・出張・オリジナル制作）の訴求として効かせる。レンタル単体は「気軽に借りられる」を前面に。

## 9. このチャットの始め方（たたき台）

1. まず「イベント・体験AR」ページから作る（最も問い合わせ単価が高い導線になりうるため）
