# サイト構成・URL設計図

> Tech Connect ハブ型サイトのページ構成、URL、相談フォームへの導線をまとめた設計図。
> 上位ルール（サイト方針・デザイン・料金）は `CLAUDE.md` を正とする。本書はページごとの詳細マップ。

---

## 1. 全体の考え方

- 流入（検索・SNS・QR・名刺）→ **トップページ（ハブ）** で信頼をつくる → 各ページへ分岐 → **全ページが無料相談フォームに集約** → 送信後は **サンクスページ** で成果（相談）を計測。
- 最優先KPIは無料相談の件数。すべての導線の最終地点はフォーム。

### 事例（実績）の扱い（決定）
- **当面は事例の独立ページ（`/works/`）を作らない。** 各サービスページ内の「実績」コーナーに統合する。
- 役割の違い：**サービスページ＝「何が頼めて、いくらか」（メニュー）／事例＝「実際にやった1件の物語」（証拠）。** 当面はメニュー側に実例を厚めに載せて兼ねる。
- 振り分け：Catch The Fruits! → イベント・体験AR、AR名刺 → 印刷物・商品AR、音羽の歌舞伎AR【要承諾】 → イベント・体験AR（相談枠の実例）。トップの実績ダイジェストにも3本を掲載。
- **昇格条件:** 1件で1ページ分の中身（写真・数字・お客様の声）を持てるほど育ったら `/works/<案件名>/` として独立ページにする。

## 4. URL・リダイレクト方針

- **トップは `/` を維持**（現行と同じ）。指名検索・既存リンクの入口を変えない。
- 新ページは上記スラッグで追加。ページ分割で旧URLを変える場合のみ、`netlify.toml`（または `_redirects`）をリポジトリに新規追加し、旧→新を 301（恒久転送）で繋ぐ。
- **QRコード・「AR体験」ボタンは別デプロイのAR体験ページへ遷移**（本サイトとは別管理）。よってページ分割の影響を受けず、リンク切れの心配はない。ただし公開前に実機で再確認する。
- 現行1ページのアンカー（`#pricing` 等）が外部で共有されている可能性は低いが、トップに同名アンカーを残せば実質維持できる。

## 5. フォーム・計測

- **フォーム:** 既存の Netlify Forms（`name="contact"`、ハニーポット）を継承。項目は最小（お名前・メール・ご相談内容）。
- **送信成功 → `/thanks/` へ遷移 → GA4のコンバージョン計測**（相談数をKPIとして測る）。
- GA4の実測定IDは未取得。サイトが形になった段階で取得・設定する。
